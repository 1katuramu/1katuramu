### About

I'm a full-stack engineer, mostly working in web dev and AI right now. Lately that's meant a lot of time in RAG pipelines and LLM apps, figuring out how to get a model to actually answer from your data instead of guessing from whatever it was trained on.

I pick up new tools/APIs fast and I'd rather have something running than spend a week planning it.

### Currently Building

A RAG chatbot. Upload a document, ask it questions, get answers pulled from that document instead of a generic LLM response.

Stack: Next.js, Postgres + pgvector, OpenAI for embeddings and chat.

Still working on retrieval quality (chunking strategy matters more than I expected). One idea I want to try once the base is solid: pointing it at something like loan or tax documents, since that's a real problem for fintechs here in Uganda.

Repo isn't public yet, will add the link once it's in decent shape.
