### About

I'm a full-stack engineer building AI-powered products, with most of my recent work sitting at the intersection of web development, machine learning, and applied LLMs. I care about the parts of AI work that don't show up in a demo: clean data pipelines, retrieval that's actually grounded, and code that still makes sense once you've shipped it and moved on.

I work daily with ChatGPT, Claude, and Gemini, not just as chat tools but as part of my actual development workflow: scaffolding code, reviewing logic, debugging, and speeding up the parts of building that don't need to be done by hand. I pair that with a solid grounding in traditional software engineering (data structures, systems, databases) so I can tell when an AI suggestion is right and when it isn't.

I learn new tools and APIs quickly, and I'd rather have a working version running than spend a week planning one. Attention to detail matters to me because AI-assisted code is only as good as the person reviewing it, and I take that part seriously.

### Currently Building

A Retrieval-Augmented Generation (RAG) chatbot. A user uploads a document, asks it questions, and gets answers pulled from that document instead of a generic model response.

**Stack:** Next.js, PostgreSQL with pgvector, OpenAI for embeddings and chat completions.

The hard part has been retrieval quality, not the LLM call itself. Chunking strategy and how you structure the vector search matter more than people expect. Once the base is solid, I want to point it at something closer to a real fintech problem: answering questions over loan terms or tax documentation, which is exactly the kind of use case African fintechs run into.

Repo isn't public yet. Will share the link once it's in a state worth showing.

### AI Tools & Workflow

I use AI tools as part of how I build, not as a novelty:

- **ChatGPT / Claude / Gemini** for scaffolding, code review, debugging, and working through design decisions before I commit to an approach
- **AI coding assistants** (Copilot-style tools) integrated directly into my editor for day-to-day development speed
- **OpenAI and Anthropic APIs** for building features, not just chatting: embeddings, structured outputs, and tool-calling patterns
- Comfortable evaluating output from these tools rather than accepting it at face value. Knowing when a suggestion is wrong is as important as knowing how to prompt well

### Machine Learning & Data Science

Working knowledge of the standard Python ML/data stack: model training and evaluation with scikit-learn, TensorFlow and PyTorch for deep learning basics, and Pandas/NumPy for data cleaning and analysis. I use this alongside RAG and LLM work rather than as a separate track: most practical AI products need both a real data layer and an LLM layer to be useful.
